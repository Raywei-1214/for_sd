# sd 图形面板说明

## 目标

Windows 侧改为通过 `PySide6` 提供图形面板，不再让用户通过控制台输入参数。图形层只负责参数收集、日志展示和状态反馈，注册核心逻辑仍复用 `batch_runner.py`。

## 当前布局

- 顶部 Hero 区：
  - 仅保留主标题 `拾米 - SD账号注册`
  - 右上角放置 `清空日志 / 打开报告目录 / 打开账号备份 / 打开日志文件 / Notion 设置` 按钮组
  - 去掉右上角工作室标识、副标题、浏览器状态徽标、Notion 状态徽标
- 左侧参数区：
  - 标题改为 `执行控制`
  - 改为与运行概览同风格的小卡片布局：每项标题在上、控件在下
  - 去掉区块说明文案，保留更干净的标题 + 表单结构
  - 注册数量
  - 并发线程
  - 邮箱站点
  - 浏览器内核（自动 / 本地 Chrome / 内置 Chromium）
  - 显示浏览器
  - 调试模式
  - Notion 开关
  - 每个邮箱站点的触发比率控制
    - 所有站点比率总和必须为 `100%`
    - 点击执行前若总和不为 `100%`，会弹窗报错并拒绝启动
  - `开始执行` 保留在第一页，放在 `启用 Notion 同步` 下方
- 第一个 Tab：`账号注册`
  - 仅保留 `执行控制`
  - 面板居中收口，不再横向铺满整页
  - 内部拆为双栏：
    - 左侧 `基础控制`
    - 右侧 `站点触发比例控制`
  - `基础控制` 与 `站点触发比例控制` 各自使用独立大底板包裹
  - `站点触发比例控制` 的数值会写入本地配置，下次打开面板会保留上次设置
- 第二个 Tab：`运行概览`
  - 左侧放 `运行概览`
  - 右侧放 `账号注册实时日记`
  - `打断结束` 挂到左侧 `运行概览` 卡片内，与运行状态同区展示
  - 顶部留白收紧，底部内容区拉开，避免“上松下挤”
- 运行概览卡片：
  - 去掉区块说明文案，仅保留统计与结果信息
  - 顶部统计卡片改为 `2x3` 布局：
    - 第一行：计划任务 / 成功 / 失败
    - 第二行：可用 / 成功率 / 可用率
  - 下方改为纵向信息块，增加实时进度条，并展示结果摘要
  - 结果摘要下方新增 `请求量/资源量统计`
    - 展示请求数、响应数、失败请求数、累计资源量与请求类型分布
  - 结果摘要下方新增 `邮箱站点质量`
    - 按每个邮箱站点逐条输出 `失败率 / 可用率 / 70积分概率 / 样本数`
    - 当某站点 `70积分概率 > 60%` 或 `失败率 > 30%` 时，该行会高亮为红色
- `账号注册实时日记` 卡片：
  - 复用 Python logger 输出
  - 补充捕获 `print` 输出
  - 成功注册日志使用绿色，Notion 写入成功日志使用橙色
  - 如果最终仍有账号没写入 Notion，日志尾部会用红字列出 `账号 / 密码 / 国家 / 错误`
  - 去掉顶部说明文案
  - `主页加载失败` 与 `临时邮箱获取失败` 会额外输出页面上下文，便于排查真实根因
- 第三个 Tab：`Dreamina 去水印`
  - 输入目录选择器
  - 运行概览卡片：计划视频 / 成功 / 失败 / 进度 / 当前视频 / 运行报告
  - 执行控制：开始去水印 / 打断结束
  - 独立去水印日志区

## 去水印能力边界

- 当前去水印能力不是通用视频去水印器。
- 仅支持 `Dreamina` 生成视频的右下角固定水印。
- 单视频时长必须 `<= 30 秒`，启动后会先做本地预检，不符合条件的批次不会进入网页处理。
- 输出目录固定为所选目录下的 `cleaned/` 子目录。
- 运行报告固定写入 `run_reports/watermark_run_<timestamp>.json`。
- 去水印与注册任务各自独立，可同时启动；关闭窗口时若仍有任一任务在执行，会提示二次确认。

## 当前按钮反馈策略

- 点击 `开始执行` 后，按钮文案会切换为 `正在执行`，并显示更强的苔绿色呼吸光晕，同时外圈高亮光柱会以更长拖尾、明显更慢的速度跑圈。
- 点击 `打断结束` 后，按钮文案会切换为 `等待收尾中`，并显示更强的陶土色呼吸光晕，同时外圈高亮光柱会以更长拖尾、明显更慢的速度跑圈。
- 任务结束、失败或清理完成后，两个按钮会恢复默认文案和静止状态。

## 当前运行概览刷新策略

- 运行开始后，批量执行器会主动向 GUI 推送进度快照。
- 完成任务时立即刷新：
  - 成功数
  - 失败数
  - 可用条数
  - 成功率
  - 可用率
  - 进度条
- 即使暂时没有新任务完成，GUI 也会按约 2 秒节奏刷新一次“运行中 / 待开始”状态。
- `可用率` 的口径与 Notion 准入规则一致：
  - 当前已完成任务中，“积分为 0、带有 sessionid、国家不含 China、且备份行以 `----0` 结尾”的账号占比
- 网络统计在批次结束后汇总展示：
  - 当前只做轻量统计，不额外发请求
  - 统计来源是浏览器上下文的 `request / response / requestfailed` 事件
- 邮箱站点质量在批次结束后刷新：
  - `失败率 = total_failure_count / attempt_count`
  - `可用率 = available_count / attempt_count`
  - `70积分概率 = credits_70_count / credits_observed_count`
  - 红字展示只负责高亮高风险站点，不改变当前用户配置的站点比例
- 左侧工作栏：
  - 恢复为普通三张卡片直排布局，不再额外包裹大面板
  - 工具按钮迁移后，左侧不再依赖滚动条

## 视觉风格

- 当前 GUI 已切换到 Organic / Natural 方向：
  - 米白纸感背景
  - 苔绿色主按钮
  - 陶土色描边按钮
  - 柔和圆角与非机械化卡片层次
  - 左侧面板改为固定宽度滚动侧栏，优先保证 Windows 下的稳定可读性
  - 字体改为接近 Codex 的简洁无衬线栈，标题与正文统一收敛
  - 主按钮改为更短更圆的胶囊圆弧形，输入框与统计卡片整体压扁一档
- 风格目标是让批量工具更像可长期使用的工作台，而不是默认系统面板。

## 默认参数

- 注册数量：`999`
- 并发线程：`5`
- 浏览器模式：隐藏
- 浏览器内核：`自动`
- Notion：开启
- 邮箱站点：`动态随机`
  - 实际界面编号会随当前站点池数量自动变化

## Notion 开关策略

- 默认开启。
- 启动前先做一次数据库连通性检查。
- 标题框右上角提供 `Notion 设置` 按钮：
  - 可直接输入 `Token`
  - 可直接粘贴数据库链接
  - 保存时自动提取 `Database ID`
  - 保存位置为运行目录下的 `.env.local`
- 若连通失败，弹窗询问是否关闭 Notion 后继续。
- 若用户确认关闭，则本次仅保留：
  - 本地 txt 账号备份
  - JSON / CSV 运行报告
  - 日志文件

## 边界

- 当前 GUI 不负责终止已启动的浏览器线程。
- `打断结束` 为软中断：
  - 已经开始执行的线程会先自然收尾
  - 尚未提交的新任务不会继续启动
- 去水印预检失败属于硬中止：
  - 不会打开 `magiceraser.org`
  - 会直接生成报告并列出失败视频与原因
- 日志文件路径采用运行时目录：
  - 源码运行时写入项目根目录下的 `dreamina_register_usa.log`
  - Windows `exe` 运行时写入 `sd.exe` 同目录下的 `dreamina_register_usa.log`
- 关闭窗口时，如果任务仍在运行，会提示二次确认。
- mac 仍维持 `.command` 调用 CLI，不强制切换 GUI。

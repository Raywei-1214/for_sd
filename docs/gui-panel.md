# sd 图形面板说明

## 目标

Windows 侧改为通过 `PySide6` 提供图形面板，不再让用户通过控制台输入参数。图形层只负责参数收集、日志展示和状态反馈，注册核心逻辑仍复用 `batch_runner.py`。

## 当前布局

- 顶部 Hero 区：
  - 仅保留主标题 `拾米 - SD账号注册`
  - 去掉右上角工作室标识、副标题、浏览器状态徽标、Notion 状态徽标
- 左侧参数区：
  - 注册数量
  - 并发线程
  - 邮箱站点
  - 显示浏览器
  - 调试模式
  - Notion 开关
- 左侧运行概览：
  - 放在运行参数下方
  - 顶部统计卡片改为 `2x2` 布局，避免在窄列里挤压
  - 下方继续展示当前状态、报告路径、结果摘要
- 左侧操作区：
  - 开始执行
  - 清空日志
  - 工具按钮改为紧凑布局，减少在左侧列里的横向拥挤
  - 打开报告目录
  - 打开账号备份
  - 打开日志文件
- 右侧实时日志：
  - 占据右侧主区域
  - 复用 Python logger 输出
  - 补充捕获 `print` 输出
  - 去掉顶部说明文案

## 视觉风格

- 当前 GUI 已切换到 Organic / Natural 方向：
  - 米白纸感背景
  - 苔绿色主按钮
  - 陶土色描边按钮
  - 柔和圆角与非机械化卡片层次
  - 左侧控制区采用更紧凑的字体和间距，优先保证 Windows 下的稳定可读性
- 风格目标是让批量工具更像可长期使用的工作台，而不是默认系统面板。

## 默认参数

- 注册数量：`200`
- 并发线程：`2`
- 浏览器模式：隐藏
- Notion：开启
- 邮箱站点：`7 - 随机`

## Notion 开关策略

- 默认开启。
- 启动前先做一次数据库连通性检查。
- 若连通失败，弹窗询问是否关闭 Notion 后继续。
- 若用户确认关闭，则本次仅保留：
  - 本地 txt 账号备份
  - JSON / CSV 运行报告
  - 日志文件

## 边界

- 当前 GUI 不负责终止已启动的浏览器线程。
- 关闭窗口时，如果任务仍在运行，会提示二次确认。
- mac 仍维持 `.command` 调用 CLI，不强制切换 GUI。

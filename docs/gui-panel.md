# sd 图形面板说明

## 目标

Windows 侧改为通过 `PySide6` 提供图形面板，不再让用户通过控制台输入参数。图形层只负责参数收集、日志展示和状态反馈，注册核心逻辑仍复用 `batch_runner.py`。

## 当前布局

- 顶部 Hero 区：
  - 项目标题
  - Aurora 风格说明
  - 浏览器状态徽标
  - Notion 状态徽标
- 左侧参数区：
  - 注册数量
  - 并发线程
  - 邮箱站点
  - 显示浏览器
  - 调试模式
  - Notion 开关
- 左侧操作区：
  - 开始执行
  - 清空日志
  - 打开报告目录
  - 打开账号备份
  - 打开日志文件
- 右侧运行概览：
  - 计划任务
  - 成功
  - 失败
  - 成功率
  - 当前状态
  - 报告路径
  - 结果摘要
- 右侧实时日志：
  - 复用 Python logger 输出
  - 补充捕获 `print` 输出

## 默认参数

- 注册数量：`200`
- 并发线程：`2`
- 浏览器模式：隐藏
- Notion：开启
- 邮箱站点：`7 - 随机`

## Notion 开关策略

- 默认开启。
- 启动前先做一次数据库连通性检查。
- 若连通失败，弹窗询问是否关闭 Notion 后继续。
- 若用户确认关闭，则本次仅保留：
  - 本地 txt 账号备份
  - JSON / CSV 运行报告
  - 日志文件

## 边界

- 当前 GUI 不负责终止已启动的浏览器线程。
- 关闭窗口时，如果任务仍在运行，会提示二次确认。
- mac 仍维持 `.command` 调用 CLI，不强制切换 GUI。

# sd 图形面板说明

## 目标

Windows 侧改为通过 `PySide6` 提供图形面板，不再让用户通过控制台输入参数。图形层只负责参数收集、日志展示和状态反馈，注册核心逻辑仍复用 `batch_runner.py`。

## 当前布局

- 顶部 Hero 区：
  - 仅保留主标题 `拾米 - SD账号注册`
  - 去掉右上角工作室标识、副标题、浏览器状态徽标、Notion 状态徽标
- 左侧参数区：
  - 改为与运行概览同风格的小卡片布局：每项标题在上、控件在下
  - 去掉区块说明文案，保留更干净的标题 + 表单结构
  - 注册数量
  - 并发线程
  - 邮箱站点
  - 显示浏览器
  - 调试模式
  - Notion 开关
- 左侧运行概览：
  - 放在运行参数下方
  - 去掉区块说明文案，仅保留统计与结果信息
  - 顶部统计卡片改为 `2x3` 布局：
    - 第一行：计划任务 / 成功 / 失败
    - 第二行：已耗时 / 成功率 / 可用率
  - 下方改为纵向信息块，增加实时进度条，并展示运行报告与结果摘要
- 左侧操作区：
  - 去掉区块说明文案，仅保留操作按钮
  - 开始执行
  - 打断结束
  - 清空日志
  - 工具按钮改为纵向按钮栈，避免横向按钮在窄列里拥挤
  - 打开报告目录
  - 打开账号备份
  - 打开日志文件
- 右侧实时日志：
  - 占据右侧主区域
  - 复用 Python logger 输出
  - 补充捕获 `print` 输出
  - 成功注册日志使用绿色，Notion 写入成功日志使用橙色
  - 去掉顶部说明文案

## 当前按钮反馈策略

- 点击 `开始执行` 后，按钮文案会切换为 `正在执行`，并显示柔和的阴影呼吸效果。
- 点击 `打断结束` 后，按钮文案会切换为 `等待收尾中`，并显示柔和的阴影呼吸效果。
- 任务结束、失败或清理完成后，两个按钮会恢复默认文案和静止状态。

## 当前运行概览刷新策略

- 运行开始后，批量执行器会主动向 GUI 推送进度快照。
- 完成任务时立即刷新：
  - 成功数
  - 失败数
  - 已耗时
  - 成功率
  - 可用率
  - 进度条
- 即使暂时没有新任务完成，GUI 也会按约 2 秒节奏刷新一次“运行中 / 待开始 / 已耗时”。
- `可用率` 的口径与 Notion 准入规则一致：
  - 当前已完成任务中，“积分为 0 且带有 sessionid”的账号占比
 - 左侧工作栏：
  - 改为固定宽度的可滚动侧栏
  - 卡片按最小高度展示，不再被右侧日志区压扁

## 视觉风格

- 当前 GUI 已切换到 Organic / Natural 方向：
  - 米白纸感背景
  - 苔绿色主按钮
  - 陶土色描边按钮
  - 柔和圆角与非机械化卡片层次
  - 左侧面板改为固定宽度滚动侧栏，优先保证 Windows 下的稳定可读性
  - 字体改为接近 Codex 的简洁无衬线栈，标题与正文统一收敛
  - 主按钮改为更短更圆的胶囊圆弧形，输入框与统计卡片整体压扁一档
- 风格目标是让批量工具更像可长期使用的工作台，而不是默认系统面板。

## 默认参数

- 注册数量：`999`
- 并发线程：`5`
- 浏览器模式：隐藏
- Notion：开启
- 邮箱站点：`7 - 随机`

## Notion 开关策略

- 默认开启。
- 启动前先做一次数据库连通性检查。
- 若连通失败，弹窗询问是否关闭 Notion 后继续。
- 若用户确认关闭，则本次仅保留：
  - 本地 txt 账号备份
  - JSON / CSV 运行报告
  - 日志文件

## 边界

- 当前 GUI 不负责终止已启动的浏览器线程。
- `打断结束` 为软中断：
  - 已经开始执行的线程会先自然收尾
  - 尚未提交的新任务不会继续启动
- 日志文件路径采用运行时目录：
  - 源码运行时写入项目根目录下的 `dreamina_register_usa.log`
  - Windows `exe` 运行时写入 `sd.exe` 同目录下的 `dreamina_register_usa.log`
- 关闭窗口时，如果任务仍在运行，会提示二次确认。
- mac 仍维持 `.command` 调用 CLI，不强制切换 GUI。
